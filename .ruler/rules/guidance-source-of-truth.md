# Guidance Source Of Truth Rule

In this repo, committed AI guidance lives in `.ruler/`. `.ruler/AGENTS.md` is the project
overview and `.ruler/<rule-name>.md` files hold focused rules; edit these by hand and commit them.
Agent-facing files at the repo root (`CLAUDE.md`, `AGENTS.md`, `.claude/`, editor rule files) are
generated by `npx @intellectronica/ruler apply` and are gitignored — never hand-edit them.

## Source Layout

- Put durable, always-on policy in `.ruler/<rule-name>.md`.
- Keep each rule focused on one concern (tooling, quality, testing, security, boundaries).
- Specs in `specs/` are product truth; rules describe how to work, not what to build.
- `.ruler/ruler.toml` controls which agent outputs are generated.

## Guidance Freshness

When current source contradicts loaded guidance, update the narrowest rule file that was wrong,
then rerun `ruler apply`. Prefer replacing stale text over appending another long note. Do not
encode one-off branch facts, temporary debugging notes, or speculative design as durable guidance.
Do not copy rules from other repos without stripping references that do not apply here.
